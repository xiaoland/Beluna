# SOP: Explore

## Role

Use when key unknowns remain and the solution space must be mapped before durable decisions or implementation.

This mode can appear inside any input type. For non-trivial code work, use implementation taste to expose authority, trust, naming, and complexity unknowns.

## Forbidden

- Do not treat provisional ideas as settled truth.
- Do not update durable docs or production code for open-ended exploration.

## Read-Do Steps

1. Open or update a task packet when the exploration is non-trivial or needs persisted context.
2. State the current unknowns and why they block progress.
3. Record only the assumptions, options, constraints, and questions that materially reduce ambiguity.
4. Identify negotiation triggers early so exploration stays bounded.
5. Exit to `Solidify`, `Execute`, or `Diagnose` once the next justified step becomes clear.

## Exit Criteria

- Unknowns are reduced enough to choose the next mode.
- Temporary assumptions and open questions are explicit.
- No provisional exploration has been promoted as durable truth.
