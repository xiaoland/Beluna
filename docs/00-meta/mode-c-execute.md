# SOP: Execute

## Role

Use when the current slice of work is clear enough to implement or edit safely.

## Forbidden

- Do not keep exploring in code when scope is still unclear.
- Do not bypass `Diagnose` when the task is really a reality mismatch.

## Read-Do Steps

1. Load the governing anchors and nearest local `AGENTS.md` files.
2. If the change is risky, reference-sensitive, or logic-altering, perform the restatement and await human confirmation.
3. Edit the smallest necessary surface area.
4. Verify the change with objective evidence.
5. Return to `Explore` or `Diagnose` if new uncertainty or unexplained behavior appears.

## Exit Criteria

- The intended change is implemented.
- Verification passes.
- Any new uncertainty has been routed into the correct next mode.
