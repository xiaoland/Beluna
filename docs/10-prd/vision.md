# Product Vision

Beluna is a survival-oriented digital life runtime, not a chatbot shell.

Beluna should:

1. Maintain coherent cognition and memory across runtime cycles.
2. Interact with world affordances through body endpoints using natural language protocol surfaces.
3. Preserve deterministic runtime behavior at module boundaries.
4. Remain operable in production with explicit observability and recovery paths.

Primary users in this phase:

- Builder/operator of Beluna runtime.
- Body-endpoint implementers integrating local or external interfaces.
