# README of docs/contract

These contrats describes behavior at a module boundary, they answers "What must be true at the boundary".

They should be:

- precise
- deterministic
- testable
- implementation-agnostic

They comes from `docs/product/features`'s user stories and acceptance criteria.
They are executable specifications for writing behavior-driven tests.
